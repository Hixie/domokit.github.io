An abstract node in a tree

AbstractNode has as notion of depth, attachment, and parent, but does not have a model for children.

Implemented by
  • RenderObject
  • RenderObject

Methods

adoptChild(AbstractNode child) → void

Subclasses should call this function when they acquire a new child.

attach() → void

Mark this node as attached.

attachChildren() → dynamic

Override this function in subclasses with child to call attach() for each child. Do not call directly.

detach() → void

Mark this node as detached.

detachChildren() → dynamic

Override this function in subclasses with child to call detach() for each child. Do not call directly.

dropChild(AbstractNode child) → void

Subclasses should call this function when they lose a child.

redepthChild(AbstractNode child) → void

Call only from overrides of redepthChildren

redepthChildren() → void

Override this function in subclasses with child nodes to call redepthChild(child) for each child. Do not call directly.
